# PROJECT RULES — Правила разработки Izborator

## 1. Принцип одного шага
В любой момент времени выполняется **один** шаг разработки.
Переход к следующему шагу — только после обновления:
- DEVELOPMENT_LOG.md
- ROADMAP_CURRENT_STEP.md

## 2. Источники истины
1. ROADMAP_CURRENT_STEP.md  
2. PLAN.md  
3. ARCHITECTURE_RULES.md  
4. MODULE_ARCHITECTURE.md  
5. DEVELOPMENT_LOG.md  

Cursor/AI обязан опираться только на них.

## 3. Запрещено
- начинать фичи, не относящиеся к текущему шагу
- менять архитектуру без прямого указания
- создавать новые файлы/папки без шага в плане
- переписывать работающий код без причины

## 4. Все изменения должны соответствовать архитектурным правилам
(см. ARCHITECTURE_RULES.md)

## 5. Никакой самодеятельности ИИ
ИИ не должен «предвосхищать» последующие шаги.

## 6. Изменения проходят 3 стадии
- обсуждение → уточнение → реализация
