# 🎯 Стратегия проекта Izborator

**Дата создания:** 2024  
**Статус:** Активная разработка

---

## 1. Миссия продукта

> **"Дать жителям Сербии быстрый и честный способ сравнивать цены и выбирать самый выгодный магазин."**

### Почему это важно
Миссия служит фильтром для всех решений — фичи, UI, архитектуры, маркетинга. Каждое техническое и продуктовое решение должно отвечать на вопрос: "Помогает ли это пользователю быстро найти лучшую цену?"

---

## 2. Цель проекта (12 месяцев)

### Метрики успеха

**Трафик:**
- 300K+ уникальных пользователей в месяц

**Данные:**
- 800K+ обновлений цен в сутки
- 150+ подключённых магазинов или парсеров

**Монетизация:**
- Доход: 3–10K€ / месяц через реферальные программы и партнёрки

## 3. Продуктовая модель

### Аналог
Izborator — это аналог E-katalog / Яндекс.Маркет для рынка Сербии. Это **не магазин**, а **система агрегации и сравнения цен**.

### Архитектура данных

#### A. Источники данных

```
┌─────────────────────────────────────┐
│  ИСТОЧНИКИ ДАННЫХ                   │
├─────────────────────────────────────┤
│  • Парсеры → собирают цены          │
│  • API магазинов (если доступно)    │
│  • Импорт CSV, XML, JSON            │
└─────────────────────────────────────┘
```

#### B. Централизованная База товаров

```
┌─────────────────────────────────────┐
│  БАЗА ТОВАРОВ                       │
├─────────────────────────────────────┤
│  • Одна карточка товара             │
│    → много магазинов                │
│  • Автоматическое сопоставление     │
│    (matching)                       │
└─────────────────────────────────────┘
```

#### C. Вывод пользователю

```
┌─────────────────────────────────────┐
│  ПУБЛИЧНАЯ ЧАСТЬ                    │
├─────────────────────────────────────┤
│  • Поиск → список товаров           │
│  • Категории → фильтры              │
│  • Карточка товара →                │
│    таблица цен + история            │
│  • График динамики → киллер-фича    │
└─────────────────────────────────────┘
```

## 4. Стратегические приоритеты

### Приоритет №1: Скорость и стабильность обновления цен
**Почему:** Без актуальных данных агрегатор теряет смысл.  
**Что делать:**
- Надёжная система парсинга с retry-логикой
- Мониторинг обновлений в реальном времени
- Приоритизация магазинов по популярности

### Приоритет №2: Точность matching товаров между магазинами
**Почему:** Неточное сопоставление делает сравнение неверным.  
**Что делать:**
- ML-модели для сопоставления товаров
- Нормализация названий, брендов, характеристик
- Ручная проверка критичных товаров

### Приоритет №3: SEO и трафик
**Почему:** 90% пользователей придут с Google, не с рекламы.  
**Что делать:**
- SSR/SSG для лучшей индексации
- Структурированные данные (Schema.org)
- Контент-страницы для категорий
- Быстрая загрузка (Core Web Vitals)

### Приоритет №4: Простая архитектура для команды из двух человек
**Почему:** Нужна система, которую легко поддерживать и масштабировать силами малой команды.  
**Стек:**
- **Backend:** Go
- **Frontend:** Next.js
- **Поиск:** Meilisearch
- **БД:** PostgreSQL
