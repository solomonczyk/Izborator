# Canonical Category Tree — v1

Цель: зафиксировать первую версию канонического дерева категорий проекта.
Версия v1 ориентирована на стабильность и расширяемость, а не на полноту.

---

## Принципы канонического дерева

1. Канон ≠ копия источника
2. Канон стабилен во времени
3. Канон понятен пользователю
4. Канон расширяется без ломки
5. Глубина ограничена (L1–L3 в v1)

---

## Структура уровней

### L1 — Domain Categories (Top-level)

L1 — это **основные пользовательские домены**, видимые:
- на Home (частично),
- в глобальной навигации,
- в SEO.

Предварительный список (v1):

- Elektronika
- Kuća i bašta
- Odeća i obuća
- Sport i rekreacija
- Lepota i zdravlje
- Deca
- Auto i moto
- Kućni ljubimci
- Knjige i zabava
- Hrana i piće
- Erotika (restricted)

---

### L2 — Core Categories

L2 — основные категории внутри домена.
Используются:
- в каталоге,
- в фильтрах,
- как точки входа.

Пример (Elektronika):

- Telefoni
- Računari
- Audio & Video
- Foto i kamere
- Gaming
- Dodatna oprema

---

### L3 — Subcategories (Leaf-level v1)

L3 — конечные категории для товаров.
Используются:
- для листинга,
- для фасетов,
- для аналитики.

Пример (Telefoni):
- Smart telefoni
- Mobilni telefoni
- Telefonska oprema

---

## Ограничения v1

- Глубина: максимум 3 уровня
- Без брендов
- Без сервисов
- Без атрибутов
- Без SEO-локализации

---

## Связь с источниками

Каждая каноническая категория:
- может иметь несколько источников
- не обязана совпадать по slug
- должна иметь объяснение происхождения

---

## Пометки и флаги

Поддерживаем флаги:
- `restricted` (Erotika)
- `needs_review`
