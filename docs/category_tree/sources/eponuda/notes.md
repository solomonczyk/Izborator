# ePonuda — Source Notes

## Источник
- URL: https://www.eponuda.com/
- Тип: marketplace / price comparison
- Регион: RS
- Язык: sr (latin)

## Что используем
- Иерархию категорий (L1 → Ln)
- Slug'и категорий
- Breadcrumb-логику (если доступна)

## Что НЕ используем
- Цены
- Товары
- Бренды
- SEO-тексты
- Внутренние ID ePonuda

## Наблюдения
- Категории ориентированы на retail
- Глубина дерева: 3–5 уровней
- Есть “пограничные” домены (Erotika, Zdravlje)
- Названия категорий — пользовательские, не технические

## Риски
- Категории могут быть избыточно детализированы
- Возможны дубли по смыслу на разных ветках
- Нет явного разделения goods / services

## Решение
- ePonuda используется как **донор структуры**, не как канон
- Финальное дерево формируется в нашем ядре
- Домены (goods/services) назначаются позже
