# ePonuda → Project Category Map

Цель: описать, как категории из ePonuda преобразуются в каноническое дерево проекта.

## Статус
- Source: ePonuda
- Raw version: tree_v1.json
- Mapping version: v1
- Тип маппинга: структурный (без семантики)

---

## Принципы маппинга

1. ePonuda НЕ является каноническим источником
2. Используется только как донор структуры
3. Канонические ID и домены назначаются в проекте
4. Одна категория источника может:
   - быть отброшена
   - быть объединена с другой
   - быть разделена
5. Никаких автоматических merge без явного правила

---

## Уровни дерева

### L1 (Top-level)
- Берём только стабильные retail-категории
- Исключаем:
  - временные
  - промо
  - дублирующие по смыслу

Примеры:
- "Elektronika"
- "Kuća i bašta"
- "Odeća i obuća"
- "Sport i rekreacija"

---

### L2 (Category)
- Используем как основу для навигации
- Возможны переименования
- Возможны объединения

Пример:
- ePonuda: "Audio-video"
- Project: "Audio & Video"

---

### L3+ (Subcategory)
- Используются как leaf-уровни
- Могут быть сокращены
- Могут быть перенесены между ветками

---

## Исключения и пограничные случаи

### Erotika
- Помечается как `restricted`
- Может быть скрыта на Home
- Доступ определяется настройками проекта/региона

### Zdravlje / Lepota
- Требует ручной проверки
- Возможное разделение goods / services позже

---

## Что НЕ делаем на этом этапе
- Не назначаем domain (goods / services)
- Не назначаем priority / weight
- Не делаем SEO-оптимизацию
- Не локализуем

---

## Output маппинга (ожидаемый)
Результат маппинга должен позволять:
- построить единое дерево категорий проекта
- добавлять другие источники (Amazon, Google, local)
- менять маппинг без изменения raw-данных
