Information Architecture Validation (v1.0)

1) Правило 3 секунд (First-3-Seconds Test)

Пользователь должен за 3 секунды понять:

 где искать (поиск в центре)

 что искать (категории вокруг)

 что произойдёт после действия (переход в каталог)

Если хоть один пункт неочевиден → упрощаем.

2) Правило одного главного действия (One Primary CTA)

На главной должен быть ровно один главный CTA:

✅ поиск (Enter / “Найти”)

Карточки — вторичный путь.

Проверка:

 визуальный вес кнопки “Найти” > любой карточки

 карточки не конкурируют с поиском цветом/контрастом

 нет второго “главного” CTA (типа “Создать аккаунт”, “Скачать”)

3) Инвентаризация элементов (UI Inventory)

Считаем всё, что видно на первом экране (desktop):

Header links: ___ (целевое ≤ 5)

Поля в поиске: ___ (целевое 1–3)

Карточки: ___ (целевое 8)

Подсказки/текст: ___ (целевое ≤ 2 строки)

Если итоговая “нагрузка” ощущается как “много всего” → режем.

4) Карточки: проверка на “визуальный шум”
4.1 Лимиты

 карточек ≤ 12

 primary карточек ≤ 4

 на каждой карточке ≤ 2 строки текста

4.2 Проверка читабельности

 карточки читаемы периферийным зрением

 пользователь понимает смысл без hover

 иконки не обязательны

5) Заголовки и текст (Copy Validation)
5.1 Заголовок (H1)

Должен отвечать: “что это за сервис”.

Примеры (шаблоны):

“Найдите товары и услуги рядом”

“Сравните предложения за минуту”

Проверка:

 H1 ≤ 8 слов

 subtitle ≤ 12 слов

 никакого маркетингового “водопада текста”

6) Navigation clarity (куда ведут карточки)

Проверка:

 при клике пользователь ожидает каталог — и получает каталог

 нет переходов на неожиданные страницы

 нет “промо-страниц” вместо каталога

7) Mobile validation (особенно важно)

На mobile критично:

 поиск виден сразу без скролла

 карточки не “облаком”, а grid

 первый экран не перегружен (≤ 1 абзац текста)

Если mobile перегружен — desktop тоже почти наверняка.

8) Accessibility validation

Проверка:

 все действия доступны без мыши

 карточки имеют focus состояния

 reduced motion не ломает смысл

9) Decision rules (что режем первым)

Если перегруз:

уменьшаем количество карточек (до 6–8)

убираем hint-тексты (оставляем только title)

убираем иконки

убираем CitySelect (если не критично)

убираем всё, кроме поиска + 6 карточек

10) Definition of Done (IA Validation)

IA считается валидированной, если:

пользователь за 3 секунды понимает сценарий

есть один главный CTA (поиск)

карточки не конкурируют с центром

mobile не перегружен

reduced motion не меняет смысл

Конец документа
