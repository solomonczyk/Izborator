UX/UI Requirements & Technical Specification (v1.0)

0. Статус документа

Тип: Обязательное ТЗ / UX-контракт

Область: Frontend / Product UX

Версия: v1.0

Статус: Draft → Implementable

Изменения: через PR + review

1. Общая концепция (Executive Summary)

1.1 Цель UX/UI системы

Создать главную страницу-дискавери, которая:

- не перегружает пользователя
- сразу даёт ощущение “я понимаю, что здесь можно найти”
- ведёт в каталог через поиск и категории, а не через баннеры

Главная страница — не витрина, а интерактивный навигатор.

1.2 Ключевая идея

Центральный блок поиска
+
Плавающие интерактивные карточки категорий, реагирующие на поведение курсора

Без:

- маркетинговых баннеров
- автоскролла
- каруселей
- “кричащих” анимаций

2. Информационная архитектура (IA)

2.1 Карта сайта (Sitemap v1)

/
├── Home / Discover
│   ├── Central Search
│   ├── Floating Category Cards
│   └── Quick entry points
│
├── /catalog
│   ├── Faceted listing (schema-driven)
│   └── Search results
│
├── /product/[id]
├── /shop/[id]           (опционально)
├── /about
├── /help
└── /legal

2.2 Роль главной страницы

Главная страница:

- НЕ отображает каталог
- НЕ содержит бизнес-логику
- НЕ знает о фасетах напрямую

Она:

- формирует навигационные намерения
- передаёт параметры в /catalog

3. Главная страница (Home / Discover)

3.1 Цели страницы

- Дать пользователю точку фокуса (поиск)
- Показать “пространство возможностей” (категории)
- Минимизировать когнитивную нагрузку

Работать одинаково хорошо:

- с мышью
- с клавиатуры
- на touch-устройствах

5. Компонентная структура (Component Map)

5.1 Layout

- Header
- HeroSearch
- FloatingCategoryCloud
- Footer

5.2 HeroSearch (центральный блок)

Назначение: точка фокуса и основной CTA

Состав:

- SearchInput
- TypeToggle (goods / services / all)
- CitySelect (если включён доменом)
- SubmitButton

Требования:

- всегда в центре экрана
- НЕ смещается при hover-анимациях карточек
- доступен с клавиатуры (Tab → Enter)

5.3 FloatingCategoryCloud

Назначение: discovery и быстрые входы

Состав:

- CategoryCard[] (6–12 штук)

Каждая карточка:

- title (категория)
- short hint (1 строка)
- icon (опционально)
- navigation intent (href)

6. Поведение карточек (Interaction Design)

6.1 Поведение мыши

Реакция на приближение курсора:

- лёгкое смещение (6–12px)
- мягкий tilt (2–4°)
- увеличение тени / контраста

Центральный блок никогда не перекрывается

6.2 Поведение при hover

- hover НЕ обязателен для понимания
- hover = усиление, а не раскрытие логики

6.3 Поведение на mobile

- hover-эффекты отключены
- карточки перестраиваются в сетку
- центральный поиск сверху
- карточки ниже (2×3 или 2×4)

6.4 Reduced Motion (обязательно)

Если prefers-reduced-motion = true:

- все движения отключены
- карточки статичны
- только цвет / тень при фокусе

7. Навигационные контракты

Карточки не содержат бизнес-логики.

Примеры:

- /catalog?type=good&category=electronics
- /catalog?type=service&category=repair
- /catalog?type=service

Канонизация:

- slug → id
- проверка доступности
- tenant-изоляция
— вне UI

8. Доступность (A11y)

Обязательные требования:

- все карточки — focusable
- Enter = переход
- Tab-навигация логична
- контраст ≥ WCAG AA
- нет hover-only информации

9. Производительность (Performance)

- ≤ 12 карточек одновременно
- только transform / opacity
- no layout thrashing
- 60 FPS на средних ноутбуках
- no JS-блокировки input’а

10. SEO и семантика

- поиск = <form>
- input = <input type="search">
- кнопка = <button>
- карточки = <a> или <button>
- H1 присутствует (скрытый допускается)

11. Definition of Done (Home v1)

Home считается готовой, если:

- поиск работает
- карточки ведут в каталог
- hover не мешает вводу
- mobile версия валидна
- reduced-motion поддержан
- lighthouse не ругается на main thread

12. Не входит в v1 (явно)

- персонализация
- ML-рекомендации
- автоскролл
- маркетинговые баннеры
- A/B тесты

13. Следующие этапы (out of scope, но зафиксированы)

- Motion v2 (более сложные взаимодействия)
- Personalization
- Analytics (heatmaps, cursor paths)
