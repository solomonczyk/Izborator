User Experience Flows (v1.0)

0) Общие принципы сценариев

Каждый сценарий:

начинается с контекста

имеет одну цель

заканчивается измеримым результатом

Нет скрытых шагов

Любой шаг должен работать:

мышью

клавиатурой

на mobile

1) Flow A — Первый визит (Desktop, мышь)
Контекст

Пользователь впервые заходит на сайт, не знает ассортимент.

Шаги

Пользователь видит центральный поиск (фокус внимания).

Периферийно замечает карточки категорий.

Двигает мышь → карточки мягко реагируют (не отвлекают).

Пользователь:

либо начинает ввод в поиск

либо наводит на карточку

Решения пользователя

Поиск → ввод запроса → Enter

Категория → hover → click

Результат

Переход в /catalog с параметрами

Пользователь понимает структуру сайта

Критерии успеха

Пользователь не ищет “где начать”

Нет паузы > 2–3 секунд перед действием

2) Flow B — Быстрый вход через категорию
Контекст

Пользователь знает, что хочет (“ремонт”, “телефоны”, “доставка”).

Шаги

Сразу кликает по карточке категории.

Происходит переход в каталог.

Фасеты уже соответствуют ожиданию.

Результат

Пользователь видит релевантный листинг

Не возвращается на главную

Критерии успеха

≤ 1 клик до релевантного контента

Нет ощущения “я попал не туда”

3) Flow C — Поиск с уточнением
Контекст

Пользователь вводит запрос, но хочет сузить область.

Шаги

Вводит запрос в поиск.

(Опционально) меняет TypeToggle.

Нажимает “Найти”.

Результат

/catalog?q=…&type=…

Фасеты подгружаются по схеме

Критерии успеха

Поиск не блокируется анимациями

Все элементы кликабельны сразу

4) Flow D — Keyboard-only (Desktop)
Контекст

Пользователь работает с клавиатуры.

Шаги

Tab → фокус на SearchInput.

Ввод текста.

Tab → кнопка “Найти” → Enter
или

Tab → карточки → Enter.

Результат

Полноценная навигация без мыши

Критерии успеха

Видимый focus-ring

Логичный порядок Tab

Нет “ловушек фокуса”

5) Flow E — Mobile (Touch)
Контекст

Пользователь зашёл с телефона.

Шаги

Видит поиск в верхней части экрана.

Скроллит вниз → видит сетку карточек.

Тап по карточке или поиск.

Отличия от Desktop

Нет hover

Нет плавающего облака

Карточки — grid

Результат

Те же точки входа, без потери смысла

6) Flow F — Reduced Motion
Контекст

Пользователь включил “уменьшить движение”.

Поведение

Нет смещений/параллакса

Только цвет/тень/outline

UX остаётся полностью понятным

Критерии успеха

Ничего “не пропадает”

Нет зависимостей от анимаций

7) Flow G — Ошибочные сценарии
7.1 Нет категорий

Карточки не отображаются

Поиск остаётся центральным

(Опционально) подсказки под поиском

7.2 Медленная сеть

Поиск доступен сразу

Skeleton вместо карточек

Нет layout shift

8) Анти-сценарии (что НЕ должно происходить)

Пользователь не понимает, куда нажать

Карточки мешают вводу

Hover необходим для понимания

На mobile “пусто” без hover

После клика ожидание без обратной связи

9) Метрики UX (v1)

Time to First Action (TFA)

Click-through-rate карточек

% пользователей, начавших с поиска vs категорий

Bounce rate главной

(без внедрения аналитики на этом этапе — только требования)

10) Definition of Done (UX Flows)

Готово, если:

каждый flow можно пройти руками

нет тупиков

desktop/mobile/keyboard покрыты

reduced-motion не ломает сценарии

UX предсказуем без обучения

Конец документа
