Home Layout Rules (v1.0)

1) Сетка и зоны экрана

1.1 Базовая структура

Экран делится на 3 зоны:

- Safe Center — зона поиска (нельзя перекрывать)
- Orbit Zone — зона карточек (плавающие карточки)
- Dead Zone — края (карточки туда не улетают, чтобы не было “по углам пусто/мешанина”)

1.2 Размеры зон (desktop)

Safe Center: прямоугольник min 520×320, max 720×420

Orbit Zone: всё вокруг Safe Center с отступом min 48px

Dead Zone: внешняя рамка экрана 24px (padding), куда карточки не заходят

2) Раскладка карточек (Desktop)

2.1 Количество

6–12 карточек одновременно

Рекомендуемо v1: 8 карточек

2.2 Позиционирование

Карточки раскладываются “кольцом” вокруг центра:

- Верхний ряд: 2–3 карточки
- Левый столбец: 1–2 карточки
- Правый столбец: 1–2 карточки
- Нижний ряд: 2–3 карточки

Правило:
ни одна карточка не пересекает Safe Center + 48px buffer.

3) Размеры карточек

3.1 Desktop размеры

width: 180–260px

height: 96–140px

radius: 16–24px

padding: 12–16px

3.2 Контент внутри карточки

Title: 16–18px, max 1 строка

Hint: 12–14px, max 1 строка

Иконка (опционально): 20–28px

4) Адаптив (Responsive)

4.1 Tablet

Safe Center уменьшается (min 420×280)

Карточек: 6–8

Раскладка: более плотное кольцо, меньше вынос в стороны

4.2 Mobile

Механика меняется:

Safe Center сверху (поиск)

Карточки превращаются в grid

2 колонки

3–6 строк (в зависимости от количества)

Никакого “облака” и hover-поведения

Тач-таргет минимум 44px высота клика

5) Загрузка и визуальная стабильность

5.1 Без скачков

Карточки должны иметь:

- предсказуемые позиции (не “рандом каждый раз”)
- анимацию появления: opacity 0 → 1 (200ms)

5.2 Скелетоны

Если категории грузятся:

показываем 6–8 skeleton cards в тех же местах

6) Порядок категорий (иерархия)

В v1 порядок фиксированный (ручной), чтобы UX был стабильный:

1–2 “главные” категории ближе к центру

менее важные — дальше

Позже (v2): персонализация.

7) Definition of Done (Layout)

Готово, если:

- центр не перекрывается ни при каких движениях
- карточки не заходят в dead-zone
- мобильная версия — сетка
- нет “случайного бардака” при перезагрузке
- скелетоны не ломают геометрию
