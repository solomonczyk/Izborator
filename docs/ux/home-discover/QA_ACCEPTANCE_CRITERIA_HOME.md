Acceptance Criteria & QA Checklist (v1.0)

1) Страница Home: базовые критерии
AC-1: Центральный поиск всегда главный

 Search блок визуально в центре на desktop

 Search блок не перекрывается карточками ни при каких hover/mousemove

 Search input кликается и получает фокус с первого клика

 Enter в input запускает переход в /catalog

AC-2: Навигация работает

 Кнопка “Найти” ведёт в /catalog с корректными query-параметрами

 Каждая карточка категории кликабельна целиком

 Клик по карточке ведёт по href без ошибок

AC-3: Нет “магии” в UI

 Карточки не “прыгают” при загрузке

 Позиции карточек стабильны между перезагрузками (если не задано иначе)

 Нет неожиданных перестроек при движении мыши

2) Карточки: interaction критерии
AC-4: Hover — усиление, не логика

 Hover изменяет только визуальные параметры (тень/scale/opacity)

 Hover не раскрывает скрытую информацию, без которой карточка непонятна

AC-5: Proximity поведение

 При приближении курсора карточка смещается мягко (<= 12px)

 Нет слежения карточки за курсором

 Нет рывков (jerk) при быстрых движениях мыши

AC-6: Active state

 При клике есть короткое подтверждение (press/scale 0.98) ≤ 120ms

 После клика всегда происходит навигация (или видимая ошибка)

3) Keyboard & A11y критерии
AC-7: Keyboard-only сценарий (обязателен)

 Tab-фокус попадает в Search input первым

 Дальше Tab идёт на TypeToggle/City (если включены)

 Дальше Tab идёт на кнопку “Найти”

 Дальше Tab идёт по карточкам категорий

 Enter на карточке = переход

AC-8: Focus-visible

 У всех интерактивных элементов видимый focus ring

 Focus ring не заменён на “нет outline”

 Focus не зависит от hover

AC-9: Контраст (WCAG AA минимум)

 Текст на карточках читаем

 Placeholder не слишком бледный

 Кнопка “Найти” читаема

4) Mobile критерии
AC-10: Mobile layout

 HeroSearch сверху

 Карточки в grid (2 колонки минимум)

 Нет hover-зависимых эффектов

 Тач-таргет ≥ 44px

AC-11: Scroll & stability

 Нет layout shift при загрузке карточек

 Нет “прыжков” при появлении контента

5) Reduced Motion критерии
AC-12: prefers-reduced-motion

 Если reduced motion включён — нет translate/rotate анимаций

 Hover/focus остаются (цвет/тень/outline)

 UX остаётся полностью понятным

6) Performance критерии
AC-13: FPS & main-thread

 На desktop нет заметных лагов при движении мыши

 Анимации только transform/opacity

 Количество карточек ≤ 12

AC-14: Input responsiveness

 Ввод в Search не лагает даже при активных hover-анимациях

7) Loading / Empty state критерии
AC-15: Loading

 Search доступен сразу

 Если карточки грузятся — показываются skeleton в фиксированных местах

 Skeleton не ломает геометрию

AC-16: Empty cards

 Если categoryCards пуст → облако не рендерится

 Нет пустого “шума” вместо карточек

8) QA Checklist (ручные проверки)
Desktop (мышь)

 Навёл на 3–5 карточек → центр не дёргается

 Быстро двигаю мышь по экрану → нет рывков и “срыва” анимации

 Печатаю в input во время движения карточек → нет лагов

Desktop (клавиатура)

 Прохожу Tab-цепочку до карточек

 Enter по карточке → переход

 Shift+Tab работает

Mobile

 Сетка карточек

 Кнопки крупные

 Нет hover-логики

Reduced motion

 Включил reduce motion → движения исчезли

9) Definition of Done (Home QA)

Home считается принятой, если:

все AC-1 … AC-16 выполнены

checklist пройден без блокеров

критические баги = 0
